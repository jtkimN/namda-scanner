# NAMDA 셀프 체크인 — 배포 가이드

> K-퀀텀 얼라이언스 포럼용 QR 체크인 시스템. 이 문서 순서대로 따라 하면 됩니다.
> 개발 지식 없이도 되도록 "화면에서 뭘 누르는지"까지 적었습니다.

---

## 0. 전체 그림 (뭐가 뭐랑 연결되나)

```
[구글폼 제출]
   │  (자동)
   ▼
[Apps Script]  → 참가자에게 번호(001~) + 보안토큰 부여, 구글시트에 기록
   │
   ▼
[뿌리오 알림톡]  →  "○○○님, 입장 QR 👉 [링크]"  (참가자 휴대폰)
   │
   ▼
[참가자가 링크 열기]  →  qr.html 이 본인 QR 표시
   │  (행사 당일)
   ▼
[태블릿 스캐너 2대]  →  index.html 이 QR 스캔 → Apps Script 로 입장 처리
```

- **구성 파일 4개**
  - `index.html` — 태블릿 스캐너 (행사장)
  - `qr.html` — 참가자 개인 QR 페이지 (알림톡 링크가 여는 페이지)
  - `generator.html` — QR 생성기 (인쇄용 명단 카드·재발급, 백업용)
  - `apps-script/Code.gs` — 구글 백엔드 (번호부여·알림톡·체크인 API)
  - `lib/` — 스캔·생성 라이브러리 (CDN 안 쓰고 내장 → 오프라인/차단에도 동작)

## 1. 준비물 체크리스트

- [ ] 구글 계정 (폼·시트 소유자)
- [ ] 뿌리오 계정 + **API 인증키** (뿌리오 > 연동 관리)
- [ ] 뿌리오에 **카카오 발신프로필** 등록 (사업자 인증 채널 연결)
- [ ] 뿌리오 **알림톡 템플릿 승인** (1~2 영업일 소요 → **미리** 신청)
- [ ] GitHub 저장소 `jtkimN/namda-scanner` (이미 있음)
- [ ] 태블릿 2대 (크롬 브라우저, 후면 또는 전면 카메라)

---

## 2. A단계 — 구글 백엔드(Apps Script) 설치

### A-1. 스크립트 열기
1. **구글폼 응답이 쌓이는 구글시트**를 엽니다. (폼 편집 화면 → 응답 → 초록 시트 아이콘)
2. 상단 메뉴 **확장 프로그램 → Apps Script** 클릭.

### A-2. 코드 붙여넣기
1. 기본 `Code.gs` 내용을 모두 지우고, 이 저장소의 **`apps-script/Code.gs` 전체**를 붙여넣습니다.
2. 💾 저장(Ctrl+S).

### A-3. 비밀값 입력 (스크립트 속성)
> ⚠️ API키·스태프키는 코드에 쓰지 않고 여기 넣습니다 (보안).

1. 왼쪽 **⚙️ 프로젝트 설정** → 아래 **스크립트 속성** → **속성 추가**.
2. 아래 표대로 입력:

| 속성 이름 | 값 | 설명 |
|---|---|---|
| `PPURIO_ACCOUNT` | (뿌리오 아이디) | 뿌리오 로그인 계정 |
| `PPURIO_APIKEY` | (뿌리오 API 인증키) | 뿌리오 연동관리에서 발급 |
| `PPURIO_SENDER` | (발신프로필 값) | 뿌리오 > 카카오 설정 > 발신프로필 관리의 값 |
| `PPURIO_TEMPLATE` | (템플릿 코드) | 승인된 알림톡 템플릿 코드 |
| `STAFF_KEY` | (직접 정한 비밀 문자열) | 스캐너에 입력할 스태프 키 (아무거나, 남에게 비공개) |
| `QR_BASE` | `https://jtkimN.github.io/namda-scanner/qr.html` | 개인 QR 페이지 주소 (C단계와 일치시킬 것) |
| `EVENT_NAME` | `K-퀀텀 얼라이언스 포럼` | (선택) |

> 문자 대체발송을 쓰려면 추가: `SMS_FALLBACK=Y`, `PPURIO_FROM=발신번호(사전등록된 번호)`.

### A-4. 초기화 실행
1. 상단 함수 선택 목록에서 **`setup`** 선택 → **▶ 실행**.
2. 처음이면 권한 승인 창이 뜹니다 → 본인 계정 선택 → "고급" → "이동(안전하지 않음)" → 허용.
   *(구글 미검증 앱 경고는 내가 만든 스크립트라 정상입니다.)*
3. 실행 후 **실행 로그**에 `setup 완료`가 뜨면 성공. 시트 오른쪽에 **번호·토큰·입장여부·입장시각·발송상태** 열이 생깁니다.

### A-5. 웹앱으로 배포 (스캐너가 부를 주소)
1. 오른쪽 위 **배포 → 새 배포**.
2. 톱니바퀴 → **웹 앱** 선택.
3. 설정:
   - 실행 계정: **나**
   - 액세스 권한: **모든 사용자** ← ⚠️ 이걸 안 하면 스캔마다 오류납니다 (지난번 주범)
4. **배포** → 나오는 **웹 앱 URL**(`.../exec`)을 복사해 둡니다. → 스캐너 게이트의 "API 주소".
5. 확인: 그 URL을 브라우저에 그냥 붙여넣어 열었을 때 `{"status":"up",...}` 가 보이면 정상.

> 🔁 **나중에 Code.gs 를 수정하면** 반드시 **배포 → 배포 관리 → (연필) 편집 → 버전: 새 버전 → 배포** 해야 반영됩니다. (URL은 그대로 유지됨)

## 3. B단계 — 뿌리오 알림톡 설정

### B-1. 발신프로필 (채널 연결)
- 뿌리오 **카카오톡 설정 → 발신프로필 관리**에서 사업자 인증 채널로 발신프로필 등록.
- 등록되면 나오는 값을 스크립트 속성 `PPURIO_SENDER` 에 입력.

### B-2. 알림톡 템플릿 등록 (승인 1~2일)
- 뿌리오 **카카오톡 설정 → 알림톡 템플릿 관리 → 등록**.
- **추천 문안** (변수 2개):

  ```
  #{var1}님, K-퀀텀 얼라이언스 포럼 참가 신청이 완료되었습니다.

  아래 링크에서 입장용 QR을 확인해 주세요.
  #{var2}

  · 일시: 2026년 O월 O일
  · 장소: 국회도서관 대강당
  · 입장 시 신분증을 지참해 주세요.
  ```
  - `#{var1}` = 성함, `#{var2}` = 개인 QR 링크.
  - Code.gs 의 `changeWord: { var1: name, var2: link }` 와 **변수 이름이 일치**해야 합니다.
    (템플릿 변수를 `#{성함}`, `#{링크}` 로 등록했다면 Code.gs 의 `sendAlimtalk_` 안 `changeWord` 키도 `성함`, `링크` 로 바꿔주세요.)
  - 링크를 **본문 텍스트**에 넣는 방식이라 버튼 설정 없이도 승인/동작이 쉽습니다. (버튼형을 원하면 웹링크 버튼 URL에 변수 사용)

### B-3. API 인증키
- 뿌리오 **연동 관리(발송 API)**에서 인증키 확인 → `PPURIO_APIKEY` 에 입력.

### ⚠️ B-4. (중요) 연동 IP 제한 확인
- 뿌리오 API에 **접속 IP 제한**이 걸려 있으면, Apps Script(구글 서버, 고정 IP 아님)에서 발송이 **막힐 수 있습니다.**
- 뿌리오 연동 설정에서 **IP 제한 없음/미사용**으로 둘 수 있는지 확인하세요.
- 만약 IP 고정이 필수라면: (1) 문자 발송으로 대체하거나 (2) 고정 IP 서버를 별도로 두어야 합니다. → 이 경우 알려주시면 대안을 만들어 드립니다.

### B-5. 발송 시험
1. Apps Script 편집기에서 `testSend` 함수 안의 전화번호를 **본인 번호**로 바꾸고 저장.
2. `testSend` 실행 → 로그에 `"ok":true` 와 함께 본인 카톡으로 알림톡이 오면 성공.
3. 이미 접수된 명단에 소급 발송하려면 `assignExisting` 실행 (인자 `true` = 알림톡도 발송). 편집기에서 `assignExisting(true)` 형태로 실행.

## 4. C단계 — GitHub Pages 배포 (스캐너·QR·생성기)

### C-1. 파일 업로드
`jtkimN/namda-scanner` 저장소에 아래를 올립니다 (기존 index.html 덮어쓰기):
- `index.html`, `qr.html`, `generator.html`
- `lib/` 폴더 (`jsQR.js`, `qrcode.js`)

> GitHub 웹에서: 저장소 → **Add file → Upload files** → 파일과 `lib` 폴더를 끌어다 놓기 → Commit.

### C-2. Pages 켜기
1. 저장소 **Settings → Pages**.
2. Source: **Deploy from a branch**, Branch: **main / (root)** → Save.
3. 잠시 후 주소가 나옵니다: `https://jtkimN.github.io/namda-scanner/`
   - 스캐너: `https://jtkimN.github.io/namda-scanner/`  (= index.html)
   - 개인 QR: `https://jtkimN.github.io/namda-scanner/qr.html`
   - 생성기: `https://jtkimN.github.io/namda-scanner/generator.html`

### C-3. 주소 일치 확인
- 스크립트 속성 `QR_BASE` 값이 **C-2의 qr.html 주소와 똑같은지** 확인. (다르면 링크가 깨집니다.)

## 5. D단계 — 태블릿 2대 세팅 (행사 당일 아침 or 리허설)

1. 태블릿 **크롬**으로 스캐너 주소(`https://jtkimN.github.io/namda-scanner/`) 접속.
2. 게이트 화면에 입력:
   - **API 주소** = A-5의 `/exec` URL
   - **스태프 키** = `STAFF_KEY` 값
   - **관리자 PIN** = 원하는 4자리 (종료·통계용)
   - **카메라** = 셀프체크인이면 전면 / 스태프가 찍으면 후면
3. **키오스크 시작** → 카메라 권한 **허용**.
4. (권장) 브라우저 메뉴 → **홈 화면에 추가** 하면 전체화면으로 뜹니다.
5. 태블릿 **자동 화면잠금 끄기**(설정) — 배터리 연결 권장.

> 관리자 메뉴: 화면 **우측 상단 모서리 5번 탭** → PIN 입력 → 통계/소리/카메라전환/종료.
> 두 대 모두 같은 API·스태프키를 쓰면 **입장 기록이 공유**되고, 한 대에서 입장한 사람을 다른 대에서 찍으면 "이미 입장함"으로 뜹니다.

---

## 6. E단계 — 리허설 테스트 (행사 전 필수 1회)

1. 폼을 **본인이 실제 제출** → 잠시 후 **알림톡 수신** 확인.
2. 알림톡 링크 열기 → 본인 QR 화면 표시 확인.
3. 태블릿 스캐너로 그 QR 스캔 → **"입장 완료 + 내 이름"** 확인.
4. 같은 QR 다시 스캔 → **"이미 입장함"** 확인.
5. 다른 태블릿으로 같은 QR 스캔 → 역시 **"이미 입장함"** (2대 공유 확인).
6. 구글시트에서 해당 행 **입장여부=입장, 입장시각** 기록 확인.

> 서버 없이 카메라·스캔만 먼저 시험: 게이트 **API 주소에 `TEST`** 입력 → 스캔하면 서버 호출 없이 인식만 확인됩니다.

## 7. 문제 해결 (FAQ)

| 증상 | 원인·해결 |
|---|---|
| 스캔하면 매번 **"서버 응답 없음"** | 웹앱 배포 액세스가 **모든 사용자**인지 확인. `/exec` 주소 정확한지, 인터넷 되는지. |
| **"확인 필요 / 등록되지 않은 QR"** | 그 참가자가 폼 제출 후 번호를 못 받았을 수 있음 → 시트에 번호·토큰 있는지 확인. `assignExisting()` 로 소급 부여. |
| **"QR이 일치하지 않습니다"** | 남의 QR이거나 옛 QR. 재발급(generator.html) 하거나 시트 토큰 확인. |
| **카메라가 안 켜짐** | 반드시 **https 주소**로 접속(카메라는 http/파일에선 안 됨). 권한 허용. 후면↔전면 전환. |
| **아이폰에서 스캔 느림** | 아이폰은 내장 jsQR 사용(정상). QR을 화면 가운데 크게, 밝기 최대. |
| **알림톡이 안 감** | B-4(IP 제한), 템플릿 승인 상태, `PPURIO_*` 속성, `testSend` 로그 확인. |
| 코드 고쳤는데 반영 안 됨 | 배포 관리 → 편집 → **새 버전** 으로 재배포. |

## 부록. QR 형식
- QR 안에 담기는 값: `NAMDA1|번호|토큰` (예: `NAMDA1|005|a1b2c3d4`)
- 스캐너는 이 `NAMDA1|` 형식만 인식하고, 번호+토큰을 서버로 보내 검증합니다.
- 일반 카메라로 찍으면 그냥 텍스트만 보입니다(정상). 반드시 **행사장 스캐너**로 찍어야 입장 처리됩니다.
